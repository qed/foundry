# Epic 1: Foundation & Mode Infrastructure (Phases 001-008)

> Database migration, mode toggle, route structure, sidebar, stage/step data model, gate check engine.
> Key files: `supabase/migrations/`, `lib/db/helix.ts`, `types/database.ts`

**001** -- Helix Mode Database Migration -- Added `project_mode` enum type ('open'|'helix') and `mode` column to the existing `projects` table (defaults to 'open'). Created `helix_steps` table with columns for step identification (stage_number, step_number, step_key), status tracking ('locked'|'active'|'complete'), flexible JSONB evidence storage (evidence_type + evidence_data), and audit trail (completed_at, completed_by). Created `helix_stage_gates` table for hard-block stage progression enforcement with status ('locked'|'active'|'passed') and audit trail. Both new tables use RLS via the existing `is_project_member()` SECURITY DEFINER helper function, plus service role bypass policies. Indexes on project_id, step_key, status, stage_number, and composite (project_id, stage_number). Unique constraints on (project_id, step_key) and (project_id, stage_number). Auto-update triggers on `updated_at` using existing `update_updated_at()` function. Created `lib/db/helix.ts` with client-side utility functions: getProjectSteps, getStepByKey, getProjectStageGates, getStageGate, updateStep, initializeHelixSteps, initializeStageGates. Updated `types/database.ts` with full Row/Insert/Update type definitions for both tables, added mode to projects type, and added convenience type aliases (HelixStep, HelixStageGate, ProjectMode, etc.). Migration files: 042_add_helix_mode.sql, 043_create_helix_steps.sql, 044_create_helix_stage_gates.sql.
