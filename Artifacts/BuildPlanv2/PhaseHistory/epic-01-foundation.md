# Epic 1: Foundation & Mode Infrastructure (Phases 001-008)

> Database migration, mode toggle, route structure, sidebar, stage/step data model, gate check engine.
> Key files: `supabase/migrations/`, `lib/db/helix.ts`, `types/database.ts`, `lib/context/helix-mode-context.tsx`, `config/helix-process.ts`, `lib/helix/gate-check.ts`, `lib/helix/deep-link.ts`, `types/helix-routes.ts`

**001** -- Helix Mode Database Migration -- Added `project_mode` enum type ('open'|'helix') and `mode` column to the existing `projects` table (defaults to 'open'). Created `helix_steps` table with columns for step identification (stage_number, step_number, step_key), status tracking ('locked'|'active'|'complete'), flexible JSONB evidence storage (evidence_type + evidence_data), and audit trail (completed_at, completed_by). Created `helix_stage_gates` table for hard-block stage progression enforcement with status ('locked'|'active'|'passed') and audit trail. Both new tables use RLS via the existing `is_project_member()` SECURITY DEFINER helper function, plus service role bypass policies. Indexes on project_id, step_key, status, stage_number, and composite (project_id, stage_number). Unique constraints on (project_id, step_key) and (project_id, stage_number). Auto-update triggers on `updated_at` using existing `update_updated_at()` function. Created `lib/db/helix.ts` with client-side utility functions: getProjectSteps, getStepByKey, getProjectStageGates, getStageGate, updateStep, initializeHelixSteps, initializeStageGates. Updated `types/database.ts` with full Row/Insert/Update type definitions for both tables, added mode to projects type, and added convenience type aliases (HelixStep, HelixStageGate, ProjectMode, etc.). Migration files: 042_add_helix_mode.sql, 043_create_helix_steps.sql, 044_create_helix_stage_gates.sql.

**002** -- Mode Context Provider & Toggle -- Created `lib/context/helix-mode-context.tsx` with HelixModeProvider wrapping Helix routes, providing useHelixMode() hook with isHelixMode, toggleMode, isLoading, currentStage, and currentStep state. Created `components/helix/helix-mode-toggle.tsx` as a client component with toggle switch, confirmation dialog for mode switching, and optional progress display (totalSteps, completedSteps, percentage). Created `app/api/projects/[projectId]/mode/route.ts` with GET (fetch current mode) and POST (update mode) endpoints — POST initializes helix_steps and helix_stage_gates rows on first switch to Helix Mode via service role client. Uses Supabase SSR auth pattern for server-side user verification.

**003** -- Helix Route Structure -- Created `types/helix-routes.ts` defining HelixStageSlug type ('discovery'|'requirements'|'architecture'|'implementation'|'testing'|'deployment'|'monitoring'|'retrospective'), HelixRouteParams interface, URL builder functions (helixRoutes.dashboard, helixRoutes.stage, helixRoutes.step), and STAGE_NUMBER_TO_SLUG mapping. Created helix layout at `app/org/[orgSlug]/project/[projectId]/helix/layout.tsx` as a client component wrapping children with HelixModeProvider and HelixSidebarWrapper, with navigation guard that redirects non-Helix projects. Created loading.tsx skeleton, page.tsx dashboard entry point, and [...slug]/page.tsx catch-all for 404 handling on unmapped sub-routes. Uses Next.js 16 async params pattern.

**004** -- Helix Sidebar & Navigation Shell -- Created `components/helix/helix-sidebar.tsx` showing 8 collapsible stages with step status icons (lock, circle, check), step counts, active/complete/locked visual states, and click navigation using helixRoutes URL builders. Created `components/helix/helix-sidebar-wrapper.tsx` as responsive wrapper with mobile hamburger toggle (Menu/X from lucide-react) and desktop fixed sidebar. Created `hooks/useStepNavigation.ts` providing goToStep, goToNextStep, goToPreviousStep functions with canGoNext/canGoPrevious flags, consuming helix-process config for step ordering.

**005** -- Stage & Step Data Model -- Created `config/helix-process.ts` as the single source of truth for the complete Helix process: 8 stages (Discovery, Requirements, Architecture, Implementation, Testing, Deployment, Monitoring, Retrospective) with 22 total steps. Defined StageConfig, StepConfig, and EvidenceRequirement TypeScript interfaces. Each step has title, description, instructions, evidence requirements (type + validation rules), and deliverables. Exported HELIX_STAGES array plus helper functions: getStage, getStep, getStageBySlug, getNextStep, getPreviousStep, getTotalSteps, isValidStepKey, getStepsForStage.

**006** -- Helix Dashboard Landing Page -- Created `components/helix/helix-stage-card.tsx` displaying individual stage cards with progress bar (completed/total steps), status badges (locked/active/complete), gate indicator, and click-through to stage detail view. Created `components/helix/helix-dashboard-metrics.tsx` showing overall progress percentage, completed/active/locked step counts, and gates passed using metric icons (CheckCircle2, PlayCircle, Lock, TrendingUp from lucide-react). Updated `app/org/[orgSlug]/project/[projectId]/helix/page.tsx` to render dashboard with metrics panel and stage card grid, consuming HELIX_STAGES config and helix step/gate data from context.

**007** -- Hard-Block Gate Check Engine -- Created `lib/helix/gate-check.ts` with core validation functions: validateEvidence (checks evidence_data against requirements from helix-process config), canCompleteStep (verifies evidence is valid before marking complete), canActivateStep (checks previous step is complete and stage gate is passed), canPassStageGate (verifies all steps in stage are complete with valid evidence). Returns GateCheckResult with passed boolean, blockers array, and warnings array. Created `app/api/helix/gate-check/route.ts` as POST endpoint accepting checkType ('complete-step'|'activate-step'|'pass-gate') with projectId and step/stage identifiers. Created `hooks/useGateCheck.ts` client hook wrapping the API with checkStepCompletion, checkStepActivation, checkStageGate methods plus isChecking/error state. Created `components/helix/gate-check-alert.tsx` rendering success/warning/error states with blocker/warning lists and dismiss callback.

**008** -- Mode Toggle UX & Open Mode Bridge -- Enhanced `components/helix/helix-mode-toggle.tsx` with progress display showing completed steps count and percentage when switching away from Helix (warns about in-progress work). Created `components/helix/project-sidebar.tsx` as dynamic sidebar that renders v1 Sidebar or HelixSidebar based on project mode from useProject context. Created `lib/helix/deep-link.ts` with HelixDeepLink interface (dashboard|stage|step types), buildHelixUrl function for generating URLs, parseHelixUrl for extracting route segments, and isHelixPath utility. Created `components/helix/deep-link-navigation-guard.tsx` that intercepts navigation to Helix URLs when project is in Open Mode (shows prompt to switch). Created `components/helix/project-breadcrumb.tsx` rendering mode-aware breadcrumbs using parseHelixUrl to extract stage/step segments, integrating with project and org contexts.
